Dual Voice Architecture
TRACK 1

VOX — REST TTS

Fire-and-forget speech via POST /v1/tts. Powers CLI/TUI voice. Emotional rewrite engine adds personality via fast model analysis of conversation context.

  • 4 modes: off / on / whisper / flat
  • Emotion engine: analyzes recent history for mood
  • Auto-detects audio player (afplay, paplay, etc.)
  • Cancelable playback (kill process + clean temp file)
  • NarrationPicker + ACK_PHRASES for tool narrations
  • Used for: notification TTS, CLI spoken responses
TRACK 2

VOICE SESSION — REALTIME WS

Full-duplex audio via wss://api.x.ai/v1/realtime. Native Grok agent — reasoning + tool calling + speech all in one stream. No sentence splitting needed.

  • VoiceSessionManager: thin audio I/O + lifecycle
  • VoiceWorker: Grok realtime agent loop
  • Server VAD: neural speech detection (auto start/stop)
  • Barge-in: cancel response on new speech
  • Audio buffering: queues chunks until WS ready
  • Used for: CoE browser Push-to-Talk

Dual-Brain Architecture
FAST PATH

GROK BRAIN

Default brain for all voice and short queries. Native voice I/O via Grok Realtime WebSocket — STT, reasoning, tool calls, and TTS all happen in a single stream with sub-second latency.

  • Native voice I/O via Grok Realtime WebSocket
  • Streaming text responses via Vercel AI SDK
  • Sub-second latency for short queries
  • Handles all voice STT → reasoning → TTS natively
  • Tool calling in both text and voice modes
  • Fallback target when Claude errors
DEEP PATH

CLAUDE BRAIN

Heavy-lift brain for complex analysis, code review, and architecture planning. Invoked via Claude Code CLI subprocess (claude -p) with full codebase access and tool use.

  • Claude Code CLI subprocess (claude -p)
  • Complex analysis, code review, architecture planning
  • 60s default timeout, retry on empty response
  • Truncation safety for large outputs
  • Falls back to Grok on error or timeout
  • Full codebase access via Claude Code tools
